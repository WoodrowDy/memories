# 동기(Synchronous) vs 비동기(Asynchronous)

> 백엔드 면접, Node.js, NestJS, 운영체제를 공부하다 보면 반드시 나오는 핵심 개념

---

## 1. 동기(Synchronous)

### 의미

- Syn(Together) + Chrono(Time)
- 시간을 맞춘다.
- 앞 작업이 끝나야 다음 작업을 시작한다.

즉,

> 결과를 받을 때까지 기다린 후 다음 작업을 수행하는 방식

### 실행 흐름

```
[A 작업 시작]
      │
      ▼
[A 작업 완료]
      │
      ▼
[B 작업 시작]
      │
      ▼
[B 작업 완료]
```

### 쉬운 비유 — 카페 줄서기

```
1번 손님 주문
      │
      ▼
커피 제조
      │
      ▼
커피 수령
      │
      ▼
2번 손님 주문 가능
```

앞 사람이 끝나기 전까지 뒷사람은 기다려야 한다.

### 코드 예시

```java
User user = userRepository.findById(id);
System.out.println(user.getName());
```

```
애플리케이션
      │
      ▼
DB 조회 요청
      │
      ▼
결과 대기
      │
      ▼
응답 수신
      │
      ▼
다음 코드 실행
```

### 장점

- 코드 흐름이 직관적
- 디버깅이 쉬움
- 순서 보장이 자연스러움

### 단점

- CPU 활용률 감소
- 처리량 감소
- 응답 지연 발생

---

## 2. 비동기(Asynchronous)

### 의미

- A(Not) + Syn(Together) + Chrono(Time)
- 시간을 맞추지 않는다.
- 나는 나, 너는 너

즉,

> 결과를 기다리지 않고 다음 작업을 수행하는 방식

### 실행 흐름

```
A 작업 시작
│
├─────────────▶ A 작업 완료
│
├─▶ B 작업 시작
│
└─▶ B 작업 완료
```

### 쉬운 비유 — 카페 진동벨

```
주문
 │
 ▼
진동벨 수령
 │
 ▼
자리 이동
 │
 ▼
다른 일 수행
 │
 ▼
벨 울림
 │
 ▼
커피 수령
```

주문 후 계속 기다리지 않는다.

### 코드 예시

```js
fetch("/users")
  .then(response => response.json())
  .then(data => console.log(data));

console.log("다른 작업 수행");
```

```
API 요청
 │
 ├────────────▶ 응답 도착
 │
 ▼
다른 작업 수행

응답 도착
 │
 ▼
결과 처리
```

### 장점

- CPU 활용률 증가
- 서버 처리량 증가
- 사용자 경험 향상

### 단점

- 코드 복잡도 증가
- 예외 처리 어려움
- 디버깅 난이도 상승

---

## 3. 왜 비동기를 사용할까?

### 핵심 이유

CPU는 빠르고 I/O는 느리다.

대표적인 I/O 작업

- DB 조회
- 외부 API 호출
- 파일 읽기
- 네트워크 통신

### 동기 방식

```
사용자 요청
      │
      ▼
DB 조회 요청
      │
      ▼
응답 대기
      │
      ▼
응답 수신
      │
      ▼
사용자 응답
```

### 비동기 방식

```
사용자 A 요청
      │
      ▼
DB 조회 요청

  사용자 B 요청 처리
  사용자 C 요청 처리

DB 응답 도착
      │
      ▼
사용자 A 응답
```

## 4. Sync ≠ Blocking

많이 헷갈리는 개념.

### Sync / Async — "결과를 언제 받을 것인가?"

### Blocking / Non-Blocking — "현재 스레드가 멈추는가?"

### 비교

```
Sync
└─ 결과를 기다림
   └─ 다음 작업 수행

Async
└─ 결과를 기다리지 않음
   └─ 다른 작업 수행
      └─ 완료 시 결과 처리
```

```
Blocking
└─ 현재 스레드 정지

Non-Blocking
└─ 현재 스레드 계속 실행
```

## 5. JavaScript는 싱글 스레드인데 어떻게 비동기를 처리할까?

### 런타임 구조

```
┌─────────────┐
│ Call Stack  │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│   Web API   │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ Callback Q  │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ Event Loop  │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ Call Stack  │
└─────────────┘
```

### 핵심

JavaScript 엔진은 싱글 스레드다. 하지만 비동기 작업은

- Web API (브라우저)
- libuv (Node.js)

가 대신 처리한다.

## 면접 단골 질문

### Q. Sync와 Blocking은 같은 개념인가?

아니다.

- Sync/Async = 결과를 언제 받을 것인가
- Blocking/Non-Blocking = 현재 스레드가 멈추는가

### Q. JavaScript는 싱글 스레드인데 어떻게 비동기가 가능한가?

Web API와 Event Loop를 사용하기 때문이다.

### Q. async/await는 동기인가?

아니다. 비동기를 동기처럼 보이게 만든 문법이다.

---

## 한 줄 정리

> 동기는 결과를 기다린 후 다음 작업을 수행하는 방식이고, 비동기는 결과를 기다리지 않고 다음 작업을 수행한 뒤 완료 시점에 결과를 처리하는 방식이다.
