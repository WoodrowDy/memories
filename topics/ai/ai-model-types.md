# AI 모델, 어떻게 나뉘나요?

> 같은 'AI 모델'이라도 **무엇을 공개하느냐**에 따라 크게 세 가지로 나뉩니다 — **폐쇄형 · 오픈 웨이트 · 오픈소스**.

## 한눈에 보기

| 구분 | 폐쇄형 (Closed) | 오픈 웨이트 (Open Weight) | 완전 오픈소스 (Open Source) |
| :--- | :---: | :---: | :---: |
| **공개 범위** | 공개 안 함<br>(API·앱으로만 사용) | 모델 가중치만 공개 | 가중치 + 학습 데이터<br>+ 학습 코드 |
| **직접 내려받기** | ❌ | ✅ | ✅ |
| **똑같이 재현·재학습** | ❌ | ⚠️<br>(데이터·코드 비공개) | ✅ |
| **대표 모델** | Claude, ChatGPT(GPT), Gemini | Llama, Gemma, DeepSeek,<br>Kimi, Qwen, Mistral | OLMo, Pythia,<br>BLOOM, Apertus |
| **만든 곳** | Anthropic, OpenAI, Google | Meta, Google, DeepSeek,<br>Moonshot, Alibaba | Allen AI(AI2), EleutherAI,<br>BigScience, Swiss AI |

## 요리에 빗대면

| 구분 | 비유 |
| :--- | :--- |
| **폐쇄형** | 완성된 요리를 **사 먹는 것**. 맛은 보지만 레시피는 모름. |
| **오픈 웨이트** | 완성된 요리를 **받아 와 집에서 데우고 변형**하는 것. 음식은 내 것이지만 레시피는 안 줌. |
| **완전 오픈소스** | **레시피와 재료까지 다 받아** 똑같이 처음부터 만들 수 있는 것. |

## 자주 하는 오해 바로잡기

> [!WARNING]
> 흔히 Llama·Gemma·DeepSeek·Kimi를 "오픈소스 AI"라고 부르지만, 엄밀히는 **가중치만 공개한 '오픈 웨이트'**입니다. 학습에 쓴 데이터와 코드는 공개하지 않아 똑같이 재현하기 어렵습니다.
>
> 데이터·코드까지 전부 공개한 **'진짜 오픈소스'**는 OLMo, Pythia, BLOOM처럼 따로 있습니다. 다만 이런 완전 공개 모델은 아직 성능 순위 최상위권에 오르는 경우는 드뭅니다.

## 가중치(weights)란?

AI가 방대한 글을 학습하며 알아낸 **패턴을 담아 둔 수십억~수천억 개의 숫자 묶음**입니다. 모델의 실력이 전부 이 숫자 안에 들어 있어, 빼버리면 빈 껍데기가 됩니다.

> [!NOTE]
> **요리 비유** — 모델 구조(설계도)가 "레시피 양식"이라면, 가중치는 "소금 몇 g·몇 도에서 몇 분" 같은 **맛을 결정하는 정확한 분량값**입니다. 오픈 웨이트는 이 분량값까지 공개해 직접 돌릴 수 있고, 폐쇄형은 비밀로 둔 채 완성된 답만 내줍니다.

## 한 줄 정리

> [!TIP]
> 공개하지 않으면 **폐쇄형**, 결과물(가중치)만 주면 **오픈 웨이트**, 만드는 과정까지 다 주면 **오픈소스**.
